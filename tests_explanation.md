# הסבר על הבדיקות במערכת (Testing Documentation)

קובץ זה מספק סקירה מלאה והסברים על מערך הבדיקות (Backend & Frontend) שהוקם עבור המערכת.

---

## 🖥️ בדיקות שרת (Backend - NestJS)

יש להריץ את הפקודות הבאות מתוך תיקיית השרת (`back/`):

1. **הרצת כל הבדיקות:**
   ```bash
   npm run test
   ```
2. **הרצת קובץ בדיקות ספציפי:**
   ```bash
   npm run test -- auth.service.spec.ts
   ```
3. **הפקת דוח כיסוי קוד (Code Coverage):**
   ```bash
   npm run test:cov
   ```

### מה נבדק בשרת?
* **שירות משתמשים (`UserService`):** יצירה, הצפנת סיסמאות, שליפה, עדכון ומחיקה מקושרת (Cascading).
* **שירות חברות (`CompanyService`):** הקמת חברה, מניעת כפילויות של שם + מזהה חברה, רישום עובדים, והיררכיית הרשאות עובדים (`OWNER`, `ADMIN`, `EDITOR`, `VIEWER`).
* **שירות אימות (`AuthService`):** הרשמה, התחברות, מחיקת משתמשי קצה רדומים, בחירת חברות אוטומטית/ידנית והנפקת טוקני JWT.

## 💻 בדיקות ממשק משתמש (Frontend - React & Vite)

בצד הלקוח הגדרנו שתי תשתיות בדיקה שונות בהתאם לסוג הבדיקה הנדרש. יש להריץ את הפקודות הבאות מתוך תיקיית הלקוח (`front/`):

### 1. בדיקות אינטגרציה ורכיבים (Vitest + React Testing Library)
מתאימות לבדיקת רכיבי UI מבודדים ולוגיקת תצוגה מקומית.
* **הרצת בדיקות האינטגרציה:**
  ```bash
  npm run test
  ```
* **הרצה במצב מעקב (Watch Mode):**
  ```bash
  npm run test:watch
  ```

#### מה נבדק?
* **רכיב קלט הטקסט (`TextInput`):**
  * רינדור תקין של ה-Label והערך ההתחלתי.
  * הפעלת פונקציית עדכון ה-State (`setState`) בעקבות אירועי הקלדה (`onChange`).
  * החלפת סוג שדה הקלט (`type="password"`) בעת לחיצה על כפתור עין התצוגה (הצגה/הסתרת סיסמה).

### 2. בדיקות מקצה לקצה (Playwright E2E)
מתאימות לבדיקת זרימות עבודה מלאות של משתמש בדפדפן אמיתי מול השרת ובסיס הנתונים (מונע תקיעות ניווטים).
* **הרצת בדיקות ה-E2E:**
  ```bash
  npm run test:e2e
  ```

#### מה נבדק?
* **זרימת הרשמה והקמה (`auth-flow.spec.ts`):**
  * כניסה לדף התחברות ומעבר לדף הרשמה.
  * הרשמת משתמש חדש (באופן ייחודי ומנוטרל התנגשויות).
  * העברה אוטומטית לעמוד הגדרת חברה (`/company-setup`).
  * הקמת חברה חדשה עם מזהה חברה.
  * העברה תקינה לעמוד הבית, הצגת הודעת ברוך הבא ופרטי החברה הפעילה.
  * התנתקות וחזרה לעמוד ההתחברות.

## 🎨 אסטרטגיות בדיקה ומוקים (Mocking)

* **בשרת:** כל הרפוזיטוריז, ה-DataSource והעסקאות (Transactions) מדומים באמצעות Mocks מלאים ב-Jest למהירות ריצה מקסימלית וללא תלות בבסיס הנתונים החי.
* **בלקוח (Vitest):** הבדיקות רצות על סביבת DOM מדומה (`jsdom`) המאפשרת לדמות דפדפן במהירות רבה ללא צורך בהרצת דפדפן אמיתי.
* **בלקוח (Playwright):** הבדיקות מריצות דפדפן Chromium אמיתי (Headless) ומדמות התנהגות משתמש קצה אמיתי, כולל תקשורת רשת אמיתית לשרת ה-NestJS המקומי.
